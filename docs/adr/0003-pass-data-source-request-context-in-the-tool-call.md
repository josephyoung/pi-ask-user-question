# Pass data source request context in the tool call

Data-source request context is supplied by the model in the `ask_user_question` tool call rather than loaded implicitly from Pi, a browser, or project configuration. `dataSourceBaseUrl` resolves relative `dataSource.endpoint` values because Pi CLI has no browser origin, and the data source may include additional request headers or cookies for authenticated APIs. These are explicit additions to the otherwise Dano-compatible model-facing parameter schema. A relative endpoint without `dataSourceBaseUrl` returns a recoverable argument error that tells the model to retry and includes correct single-question and grouped-question parameter examples. Once a form is active, data-source transport or parsing failures appear on the affected field and leave the form open so the user can retry without terminating the tool call.
